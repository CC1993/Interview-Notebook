# 学习

## 怎么学习

对于一个新技术，我通常根据需要使用该技术要解决的问题，选择性地找一些资料进行学习，并且试着写一个 Demo 程序。在碰到问题时再使用 Google 找解决方法，并不断地深入学习。之后，会找一些比较靠谱的技术书籍进行系统地学习。

## 技术书籍

- Java：Java 编程思想、Effective Java、Java 并发编程实战、深入理解 Java 虚拟机、Spring 实战
- 数据库：高性能 MySQL、MySQL 技术内幕、Redis 设计与实现、Redis 实战
- 网站架构：大规模分布式存储系统、大型网站技术架构、大型网站系统与 Java 中间件开发实践、

# 性格

## 优缺点

优点：

- 学习能力不错，比如高中和大学的考试，复习一个多星期就能考得比较好的成绩。
- 善于将工作任务进行分解，从而更高效完成任务，比如平时一直在用滴答清单这种任务管理软件，当有一个工作或者学习任务的时候，先认真地将任务分解成一个个容易完成的小任务，再去逐个完成。
- 有记录总结的习惯，比如平时在学习或者工作的过程中，会使用笔记软件对一些重要内容进行认真总结记录。
- 沟通能力

缺点：

- 性子比较急，对于一个需要完成的任务，没有做完的话会很不自在，所以为了完成一些任务经常不按点吃饭，熬夜，对自己的身体特别不好。

> 初试时提出的问题最好少涉及薪金、待遇，而应询问有关工作职责、业务范畴之类，使用人单位感受到求职者的敬业精神；而在复试时则可以讨论诸如薪酬福利、交通、培训等同个人利益比较相关的问题。但切记，问到个人待遇方面的问题要谨慎适度，用人单位介绍过的就不必多问，也不要喋喋不休反复问个不停，更不能表现出算帐本领高超，十分精明。 三是澄清。每个求职者都应当确信通过数次面试下列问题已是心中十分明了的，如果心中无数，则一定要问清。这些问题主要包括：用人单位规模、求职者的职务与职责、技术与设备水准、产品的水平、市场占有率、用人单位的发展目标（即招人的动机）、求职者所处的部门的纵向（上、下级）和横向（其它部门）的关系、薪资待遇、其他福利等。如果求职者尚未搞清上述的全部问题而不提问或只在某一二个问题上反复计较都是不理智的行为。

# 阿里

## 请介绍一项你最热爱、最擅长的专业领域，并且介绍你在这个领域的学习经历和未来的学习规划。

Java 研发。在我本科还没有多少编程经验的时候，为了能多学点东西，就找了教我数据结构课程的老师，表示想进入他的实验室多点实践经验。老师热情地欢迎我的加入，并让一个学长指导我，安排给我一些简单的数据处理工作。从那时候刚开始学习 Java，算是我的编程入门。简单了解一点语法之后，顺利地完成安排给我的工作。之后陆陆续续接触一些比较复杂的内容，例如从各种文档中提取出文本信息，中文文本分词，相似度分析等。从那时候我就发现自己对技术比较感兴趣，而且自己的性格也比较适合技术相关的研究工作。再后来，实验室接了一个和中交二公院的合作项目，内容是对公路设计的安全性分析，我也参与到其中，主要负责一些 CAD 绘图和出表工作。因为这个项目需要实际使用，不仅功能复杂，而且对代码风格和 UI 规范等有很高的要求。完成这个项目之后，自己对软件设计和代码规范有了新的认识。因为一些学业上的原因离开了那个实验室，不过我对技术研究的热情没有减退。后来买了很多技术书籍来充实自己，并尽可能地实现书中的内容来提高对技术的理解程度。于此同时发现了 Leetcode 这个 OJ 网站，陆陆续续在上面做了四百多算法题，有很大的收获。因为课程的需要，本科阶段也写了一些课程小项目，比如论坛系统的实现，仓库管理系统的实现，使用 Lucene 实现了一个搜索系统。到了研究生阶段，开始接触一些图形学相关的研究工作。因为之前没有相关的知识，所以很多内容都要重新学起。在这一期间，主要实现了以下工作：心电图分类器、CT 图像三维重建、密集点云的四边形网格生成。完成这些内容踩了很多坑，例如做了很长时间之后被导师推翻了之前的工作成果，要求重新做。除了以上工作，最近我还正在学习分布式相关的内容，我觉得分布式的技术特别酷，它让一个本来无法实现的大型服务得以实现。因此，我期望能够从事分布式相关的工作，并且也会不断地学习分布式的知识。

## 请介绍你参与的印象最深刻的一个项目，为什么？并且介绍你在项目中的角色和发挥的作用。

CT 图像三维重建项目。这是我研究生的科研项目，因为之前没有太多的图形学知识，因此在开始这个项目的时候，查阅了很多资料来补充相关知识，例如 OpenGL 编程，网格操作，样条曲线，三维重建技术等等。而且实验室之前没人做这个方向，因此需要自己去填坑。这个过程很艰辛，实现一个内容之后突然发现想法完全错误，或者是实现得差不多了但是导师不认可。经过了很多次尝试之后，目前这个项目已经基本完成，正在写论文近期准备投稿。这个项目是我和一个博士师兄一起合作的，我主要负责实现，师兄主要负责寻找研究点。不过由于我们俩对这个方向都不熟悉，因此我也是需要一起找资料，并且研究点往往是在实现的过程中产生，需要去寻找论文验证自己的想法的正确性。
